Tokens & Authentication

Every API call requires an access token. There are two kinds. Using the wrong one is the single most common cause of a bot that works for a day and then mysteriously stops.

The two token types

Temporary TokenSystem User Token
Lifespan~24 hoursNever expires (until manually revoked)
Where to get itDeveloper App → WhatsApp → API Setup → "Generate"Meta Business Manager → System Users
Good forInitial testing onlyProduction / anything running unattended
Expiry symptomBot receives messages but never replies; logs show OAuthException code: 190N/A — never expires
Token expired — what it looks like

If your bot receives incoming messages (you can see the POST requests in your server logs) but never sends a reply, and your logs show something like:

JSON
{
  "error": {
    "message": "Invalid OAuth access token.",
    "type": "OAuthException",
    "code": 190
  }
}

Your temporary token has expired. Fix: either regenerate a new temporary token (short-term) or set up a System User Token (the right solution).

1

Getting a temporary token (quick start)

  1. Go to developers.facebook.com/apps → your app → WhatsApp → API Setup
  2. Under "Access Tokens", click Generate
  3. Copy the token (starts with EAA)
  4. Set it as your WHATSAPP_TOKEN environment variable

This token expires in approximately 24 hours. Only use it for local development — never in a deployed server.

2

Create a System User (for production)

  1. Go to business.facebook.com
  2. Click the gear icon → Business Settings
  3. In the left menu under "Users", click System Users
  4. Click Add
  5. Give it a name (e.g. "WhatsApp Bot") and set the role to Admin
  6. Click Create System User

The System User is now created but has no permissions. The next two steps add them.

3

Add the System User to your app

  1. Still in System Users, click your new user
  2. Click Add Assets
  3. Under "Apps", find and select your WhatsApp Developer App
  4. Toggle on Manage app
  5. Click Save Changes
4

Generate a permanent token

  1. In System Users, click your System User
  2. Click Generate New Token
  3. Select your WhatsApp app from the dropdown
  4. Check these two permissions:
    • whatsapp_business_messaging
    • whatsapp_business_management
  5. Set the token expiry to Never
  6. Click Generate Token
  7. Copy the token immediately — Meta will not show it again
Set expiry to Never

If you set an expiry date during generation, you defeat the purpose of using a System User. Make sure the expiry is set to Never.

5

Store the token safely

Your token is a secret credential. Treat it like a password.

.env 
# NEVER commit this file to Git
WHATSAPP_TOKEN=EAAVGz...
PHONE_NUMBER_ID=1043096775563494
VERIFY_TOKEN=your_chosen_webhook_secret
  • Add .env to your .gitignore file
  • For deployed servers: use the platform's secrets/environment variables UI (Render, Cloud Run, Railway, Heroku all have one)
  • Never hardcode tokens directly in source code
Accidentally pushed to GitHub?

If you accidentally commit and push a token to a public (or even private) GitHub repo, revoke it immediately: Business Settings → System Users → your user → revoke the token. Then generate a new one. GitHub scans public repositories for secrets and may notify Meta automatically — but don't wait for that, revoke it yourself right away.
