Phone Number Registration

Meta provides a free test number for development. When you're ready for production, you'll register your own number. This section covers both, plus how to set up Postman with the official Meta collection.

1

Using the test number

In WhatsApp → API Setup, Meta provides a test phone number (usually a US +1 number). It's ready to use immediately — no registration needed.

To add recipients your test number can message:

  1. In API Setup, under the "To" section, click "Manage phone number list"
  2. Click Add phone number
  3. Enter the number and verify it via WhatsApp OTP
  4. Repeat for up to 5 numbers

Once added, you can send messages to those numbers using the API.

2

Set up Postman with Meta's official collection

Postman is the best tool for making the API calls needed to register your number and subscribe your WABA. Meta maintains an official Postman workspace with all their API calls pre-built.

  1. Go to postman.com/meta
  2. Find the WhatsApp Business Platform collection
  3. Click Fork (top right of the collection) — this copies it to your own Postman account
  4. In your fork, click on the collection name and go to the Variables tab
  5. Set these collection variables:
VariableValue
access_tokenYour temporary or permanent token (see Section 7)
phone-number-idYour Phone Number ID from API Setup
business-account-idYour WABA ID from API Setup
versionv22.0
Always fork, never just open

A fork gives you your own copy where you can save variable values and response history. Opening the collection directly means your changes aren't saved between sessions.

3

Phone number rules — what is and isn't allowed

Not every phone number can be used with WhatsApp Cloud API. Before you try to register, check your number against these requirements:

RuleDetail
Must be able to receive SMS or voice calls Meta sends an OTP to verify the number. It must be reachable via SMS or automated voice call at the time of registration.
Cannot be currently active in any WhatsApp app If the number is registered in WhatsApp (personal or WhatsApp Business App), you must migrate it out first. A number cannot exist in two WhatsApp systems at the same time.
Cannot be a short code Short codes (e.g. 5-digit numbers) are not supported. You need a standard local or international number.
Cannot be a toll-free number in all regions Toll-free numbers are not supported in all countries. Check the Meta docs for your country.
Must be a number you own and control You will need to receive an OTP on it during setup, and you'll be responsible for the number going forward.
Landlines are supported Landlines can be used if they can receive an automated voice call for the OTP. Select "Call me" instead of SMS during verification.
The number's country must be supported Most countries are supported but check Meta's country list for any restrictions in your region.
Number already active in WhatsApp? Migration required

If the number is currently active in the consumer WhatsApp app or the WhatsApp Business App, you must delete the WhatsApp account on that app before Meta will allow the number to be added to the Cloud API. This process deletes all message history and backup data on that device. There's no way around it — one number, one system.

To delete: open WhatsApp → Settings → Account → Delete my account. After deletion, wait a few minutes before trying to add the number in Meta Business Suite.

4

Add your number in Meta Business Suite

Once you've confirmed the number is eligible:

  1. Go to business.facebook.com → Settings → WhatsApp Accounts
  2. Click the WABA you want to use
  3. Under Phone numbers, click Add phone number
  4. Enter your number in international format (e.g. +263775550000)
  5. Choose verification method: Text message or Voice call
  6. Enter the 6-digit OTP you receive
  7. Your number now appears in the list with its own Phone Number ID — copy it
5

Register the number via Postman (the API call)

Adding the number in Business Suite is only half the process. You must also register it via the API to activate it for Cloud API messaging. This is where Postman comes in.

Step-by-step in Postman:

  1. Open your forked Meta Postman collection
  2. In the collection's Variables tab, make sure phone-number-id is set to your new number's Phone Number ID (not the test number's ID)
  3. Also make sure access_token is current and valid
  4. Find the request named "Register Phone Number" in the collection (under the Phone Numbers folder)
  5. If you can't find it, create a new POST request manually:
HTTP 
POST https://graph.facebook.com/v22.0/{{phone-number-id}}/register
Authorization: Bearer {{access_token}}
Content-Type: application/json

{
  "messaging_product": "whatsapp",
  "pin": "123456"
}

The pin is a 6-digit number you choose. This becomes the two-step verification PIN for your number. Write it down — you'll need it if you ever need to re-register or migrate the number.

Expected success response:

JSON 
{ "success": true }

Common error responses and what they mean:

ErrorCauseFix
code: 190 Access token is expired or invalid Generate a new token (Section 7)
code: 100 — "Invalid parameter" Wrong Phone Number ID in the URL, or the number wasn't added in Business Suite first Double-check the Phone Number ID; confirm the number exists in Business Suite
"Phone number is already registered" The number was previously registered (maybe through the WhatsApp Business App) Deregister first, then re-register; or migrate the number out of the old system
code: 131031 Two-step verification is already enabled with a different PIN You need to disable two-step verification first via the Business Suite settings, then re-register with your new PIN
6

Subscribe your WABA to your app (critical)

This is the same call mentioned in Section 2, but it bears repeating because it's so commonly missed. After adding your permanent number, its WABA must be subscribed to your Developer App. Without this, messages to your permanent number will never reach your webhook — even though the dashboard may show everything as connected.

Run this directly in your terminal, or use Postman — whichever is faster when you're troubleshooting:

curl (terminal) 
curl -X POST "https://graph.facebook.com/v15.0/{WABA_ID}/subscribed_apps" -H "Authorization: Bearer {WHATSAPP_TOKEN}"

Or in Postman as a POST request:

HTTP 
POST https://graph.facebook.com/v22.0/{{business-account-id}}/subscribed_apps
Authorization: Bearer {{access_token}}

No request body needed for either. Expected response:

JSON 
{ "success": true }
curl is the fastest way to debug this

If messages still aren't arriving after going through the Postman setup, paste the curl command straight into your terminal with your real WABA ID and token. You get an immediate response and can rule out any Postman variable misconfiguration as the cause.

Once you get { "success": true } from both the register call and this call, your number is fully active and connected. Incoming messages will flow to your webhook.

7

Verify the number is active — send a test message

The final check: use Postman to send a message from your new number to one of your test recipients. Find "Send Message" in the Meta Postman collection, or use this request:

HTTP 
POST https://graph.facebook.com/v22.0/{{phone-number-id}}/messages
Authorization: Bearer {{access_token}}
Content-Type: application/json

{
  "messaging_product": "whatsapp",
  "to": "263775550000",
  "type": "text",
  "text": { "body": "Test message from my registered number" }
}

If the message arrives on the recipient's WhatsApp, your number is fully registered and working. If you get an error, the table above will help diagnose it.
