Message Types

The WhatsApp Cloud API supports several distinct message types — each with its own payload structure, character limits, and rules about when you can use it. This page covers all of them.

Overview — when can you send each type?

Message TypeRequires template?When allowed
TextNoWithin 24h customer service window
Media (image, video, document, audio)NoWithin 24h customer service window
Interactive — Reply ButtonsNoWithin 24h customer service window
Interactive — ListNoWithin 24h customer service window
Template (marketing / utility)Yes — pre-approvedAny time, including to start a conversation
Authentication Template (OTP)Yes — auto-approvedAny time
24-hour window rule

Once a user messages you, you have a 24-hour window to reply with any message type — no template needed. After that window closes, you can only contact them again using a pre-approved template. This is why templates matter for notifications and re-engagement.

1. Text messages

The simplest type. Plain text, no template required, works within the 24-hour window.

FieldLimit

3. Interactive — Reply Buttons

Buttons appear as tappable options below your message. The user taps one and it sends their selection back to your webhook as an interactive message.

ComponentLimit
Number of buttonsMax 3
Button title (label text)Max 20 characters
Button ID (your identifier for the tap)Max 256 characters
Body textMax 1024 characters
Header (optional)Text (max 60 chars), or image, video, document
Footer (optional)Max 60 characters
Use the ID, not the title, in your logic

When a user taps a button, your webhook receives interactive.button_reply.id and interactive.button_reply.title. Always check the id in your code — never the title. Titles can change; IDs are what you control and what your logic should key off.

JSON (send buttons)
{
  "messaging_product": "whatsapp",
  "to": "263776001234",
  "type": "interactive",
  "interactive": {
    "type": "button",
    "header": {
      "type": "text",
      "text": "Your order is ready"
    },
    "body": {
      "text": "Order #1042 is packed and waiting. What would you like to do?"
    },
    "footer": {
      "text": "Reply within 24 hours"
    },
    "action": {
      "buttons": [
        {
          "type": "reply",
          "reply": { "id": "confirm_pickup", "title": "Confirm Pickup" }
        },
        {
          "type": "reply",
          "reply": { "id": "reschedule", "title": "Reschedule" }
        },
        {
          "type": "reply",
          "reply": { "id": "cancel_order", "title": "Cancel Order" }
        }
      ]
    }
  }
}
JSON (incoming button tap — from your webhook)
{
  "type": "interactive",
  "interactive": {
    "type": "button_reply",
    "button_reply": {
      "id": "confirm_pickup",
      "title": "Confirm Pickup"
    }
  }
}

4. Interactive — List Messages

A list shows a scrollable menu when the user taps a button. Better than buttons when you have more than 3 options.

ComponentLimit
Total rows (items)Max 10 across all sections
Number of sectionsMax 10
Row titleMax 24 characters
Row description (optional)Max 72 characters
Row IDMax 200 characters
Section title (optional)Max 24 characters
Button label (the "tap to open list" text)Max 20 characters
Body textMax 1024 characters
Header (optional)Text only, max 60 characters
Footer (optional)Max 60 characters
10 rows total, not 10 per section

The limit is 10 rows spread across all sections combined — not 10 per section. If you have 2 sections with 5 rows each, that's your maximum. Plan accordingly when designing your menus.

5. Message Templates (Marketing & Utility)

Templates are pre-written messages approved by Meta. You use them to start conversations or send notifications outside the 24-hour window. They are not free-form — the structure is fixed and must match exactly what Meta approved.

Template components

ComponentRequired?Limits
HeaderOptionalText (max 60 chars) or media (image, video, document)
BodyRequiredMax 1024 characters; variables allowed: {{1}}, {{2}}, etc.
FooterOptionalMax 60 characters; no variables
ButtonsOptionalSee button types below

Button types in templates

Button typeMax countWhat it doesChar limit
Quick Reply Max 3 User taps → sends a reply back to your webhook; you handle it like a button_reply 25 chars per button
Call to Action — URL Max 2 Opens a URL in the browser; can include a dynamic variable in the URL path 20 chars label / 2000 chars URL
Call to Action — Phone Max 1 Opens a dial prompt with a preset phone number 20 chars label
Copy Code Max 1 Copies a code (OTP/coupon) to clipboard when tapped — used in Authentication templates N/A
Quick Reply and Call to Action cannot be mixed

A template can have up to 3 Quick Reply buttons or up to 2 Call to Action buttons — but not both types in the same template. Choose one category of button per template.

JSON (sending a template with variables)
{
  "messaging_product": "whatsapp",
  "to": "263776001234",
  "type": "template",
  "template": {
    "name": "order_confirmation",
    "language": { "code": "en_US" },
    "components": [
      {
        "type": "header",
        "parameters": [
          { "type": "image", "image": { "link": "https://example.com/logo.png" } }
        ]
      },
      {
        "type": "body",
        "parameters": [
          { "type": "text", "text": "John" },
          { "type": "text", "text": "ORD-9921" },
          { "type": "text", "text": "2 business days" }
        ]
      },
      {
        "type": "button",
        "sub_type": "quick_reply",
        "index": "0",
        "parameters": [{ "type": "payload", "payload": "track_ORD-9921" }]
      }
    ]
  }
}

The body of this template was approved as: "Hi {{1}}, your order {{2}} will arrive in {{3}}.". The three parameters fill in {{1}}, {{2}}, {{3}} in order.

Creating a template

  1. Go to business.facebook.com → your WABA → Message Templates
  2. Click Create Template
  3. Choose category: Marketing (promotions), Utility (transactional — order updates, receipts), or Authentication (OTP)
  4. Write the body, add optional header/footer/buttons
  5. Submit for review — Utility and Authentication templates are usually approved within minutes; Marketing can take longer

6. Authentication Templates (OTP / password prompt)

Authentication is a special template category designed specifically for sending one-time passwords, verification codes, and login prompts. Meta auto-approves these far faster than marketing templates because the use case is clear and tightly scoped.

FeatureDetail
CategoryAuthentication
Approval speedUsually auto-approved within minutes
Body variableOne variable {{1}} for the code itself
Code expiry noticeOptional — you can set an expiry time (e.g. "This code expires in 10 minutes")
ButtonOne Copy Code button — user taps it and the code is copied to their clipboard automatically
Header / FooterNot allowed on Authentication templates
JSON (sending an OTP via Authentication template)
{
  "messaging_product": "whatsapp",
  "to": "263776001234",
  "type": "template",
  "template": {
    "name": "otp_verification",
    "language": { "code": "en_US" },
    "components": [
      {
        "type": "body",
        "parameters": [
          { "type": "text", "text": "847291" }
        ]
      },
      {
        "type": "button",
        "sub_type": "url",
        "index": "0",
        "parameters": [
          { "type": "text", "text": "847291" }
        ]
      }
    ]
  }
}

The template body is typically something Meta generates based on your app name: "{{1}} is your [App Name] verification code." The Copy Code button lets the user copy 847291 directly to their clipboard without manually selecting it.

Authentication templates are free

Authentication conversations are billed at a lower rate than marketing conversations in most regions, and in many countries they fall under the free tier. Check the Meta pricing page for your region.
