Step 8 of 12

Pricing

WhatsApp Cloud API pricing is conversation-based, not per-message. Understanding this model before you build is essential for planning costs — especially if your users will be initiating conversations vs. you reaching out to them.

1

How conversation pricing works

Meta charges per conversation, not per message. A conversation is a 24-hour window that opens when the first message is sent between you and a user.

Any number of messages exchanged within that 24-hour window counts as a single conversation. A new window — and a new charge — opens only when a message is sent after 24 hours of inactivity.

2

The two conversation types

How a conversation starts determines which category it falls into — and what you pay:

User-Initiated (Service)Business-Initiated
Triggered by The user messages you first You send the first message to the user
Requires a template? No — you can reply with free-form text Yes — the opening message must be a pre-approved template
Cost Lower rate (varies by country) Higher rate (varies by country and category)
Free tier 1,000 conversations/month free No free tier
24h reply window You have 24 hours to reply freely after the user messages you You open the window by sending a template
The free tier is per WABA, not per number

The 1,000 free user-initiated conversations per month is shared across all phone numbers in one WABA. If you have multiple numbers under the same WABA, they share that allowance. The free tier resets on the 1st of each month.

3

Template messages — when you need them

A template message is a pre-approved message format that you must use when you initiate a conversation (business-initiated). You cannot send a free-form text message to a user who hasn't messaged you first within 24 hours.

Templates have three categories:

  • Marketing — promotions, offers, announcements (highest cost)
  • Utility — order confirmations, delivery updates, appointment reminders (moderate cost)
  • Authentication — OTPs, verification codes (lowest cost)
4

Creating and approving a template

  1. Go to business.facebook.com → Settings → WhatsApp Accounts → your WABA
  2. Click Message Templates → Create Template
  3. Select a category (Marketing / Utility / Authentication)
  4. Write the template — use {{1}}, {{2}} etc. for variable placeholders
  5. Add a sample value for each variable (required by Meta for review)
  6. Submit for approval

Utility and Authentication templates typically approve within minutes to hours. Marketing templates can take hours to days.

5

Sending a template message — code example

Once your template is approved, you send it like this:

JavaScript 
const axios = require('axios');

await axios.post(
  `https://graph.facebook.com/v22.0/${process.env.PHONE_NUMBER_ID}/messages`,
  {
    messaging_product: 'whatsapp',
    to: '263775550000',       // recipient: international format, no + or spaces
    type: 'template',
    template: {
      name: 'hello_world',    // your approved template name
      language: { code: 'en_US' },
      components: [
        {
          type: 'body',
          parameters: [
            { type: 'text', text: 'John' }   // replaces {{1}} in the template
          ]
        }
      ]
    }
  },
  {
    headers: {
      Authorization: `Bearer ${process.env.WHATSAPP_TOKEN}`,
      'Content-Type': 'application/json'
    }
  }
);
Phone number format

The to field must be in international format without the + sign or any spaces or dashes. For example: Zimbabwe (+263) 77 555 0000 becomes 263775550000.

6

Actual pricing

Rates vary significantly by the recipient's country. For example, conversations to users in Zimbabwe, South Africa, Kenya, and other African countries have different rates than conversations to users in the US or Germany.

Check the official pricing page

Meta publishes a full per-country rate table. Always check the current rates before planning your pricing model — rates change and the table is the authoritative source.
