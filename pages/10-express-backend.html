Simple Express Backend

Here is a minimal but complete Express server that handles webhook verification, receives incoming messages, and sends replies. Everything you need to get the API talking to your code.

What your server needs to do

  1. Answer Meta's webhook verification — a one-time GET request Meta sends to confirm your server is real
  2. Receive incoming messages — a POST request Meta sends whenever a user messages your bot
  3. Send replies — an outgoing POST from your server to the WhatsApp API

Complete index.js

JavaScript (index.js) 
require('dotenv').config();
const express = require('express');
const axios   = require('axios');
const app     = express();

app.use(express.json());

const PORT = process.env.PORT || 3000;

// Webhook verification — Meta sends this once to confirm your server is reachable
app.get('/webhook', (req, res) => {
  const mode      = req.query['hub.mode'];
  const token     = req.query['hub.verify_token'];
  const challenge = req.query['hub.challenge'];

  if (mode === 'subscribe' && token === process.env.VERIFY_TOKEN) {
    console.log('Webhook verified successfully');
    res.status(200).send(challenge);
  } else {
    res.sendStatus(403);
  }
});

// Receive incoming messages
app.post('/webhook', (req, res) => {
  // Respond 200 immediately — Meta requires a response within 5 seconds
  res.sendStatus(200);

  const body = req.body;
  if (body.object !== 'whatsapp_business_account') return;

  const entry         = body.entry?.[0];
  const change        = entry?.changes?.[0];
  const value         = change?.value;
  const messages      = value?.messages;

  if (!messages || messages.length === 0) return; // status update, not a message

  const message       = messages[0];
  const from          = message.from;           // sender's phone number
  const phoneNumberId = value.metadata?.phone_number_id;

  console.log(`Incoming message from ${from}:`, JSON.stringify(message));

  // Route by message type
  if (message.type === 'text') {
    const text = message.text.body;
    sendTextMessage(phoneNumberId, from, `You said: ${text}`);
  } else if (message.type === 'interactive') {
    handleInteractive(phoneNumberId, from, message.interactive);
  }
  // Add more type handlers as needed: 'image', 'document', 'location', etc.
});

async function sendTextMessage(phoneNumberId, to, text) {
  try {
    await axios.post(
      `https://graph.facebook.com/v22.0/${phoneNumberId}/messages`,
      {
        messaging_product: 'whatsapp',
        to,
        type: 'text',
        text: { body: text }
      },
      {
        headers: {
          Authorization: `Bearer ${process.env.WHATSAPP_TOKEN}`,
          'Content-Type': 'application/json'
        }
      }
    );
  } catch (err) {
    console.error('Send error:', err.response?.data || err.message);
  }
}

function handleInteractive(phoneNumberId, from, interactive) {
  if (interactive.type === 'button_reply') {
    const buttonId    = interactive.button_reply.id;
    const buttonTitle = interactive.button_reply.title;
    sendTextMessage(phoneNumberId, from, `You clicked: ${buttonTitle} (id: ${buttonId})`);
  } else if (interactive.type === 'list_reply') {
    const listId = interactive.list_reply.id;
    sendTextMessage(phoneNumberId, from, `You selected: ${listId}`);
  }
}

app.listen(PORT, () => console.log(`Server running on port ${PORT}`));
Always respond 200 before processing

Meta requires your webhook to respond with HTTP 200 within 5 seconds. If your processing (database queries, external API calls) might take longer, respond with 200 first and process asynchronously afterward. If Meta doesn't get a 200 in time, it retries the delivery — which can cause duplicate message handling.

Setting up the webhook in Meta

1

Deploy your server

Your server must be publicly accessible over HTTPS with a permanent URL. The easiest way is to connect your GitHub repository to a free hosting platform:

  • Render — connect GitHub repo, select "Web Service", free tier. URL: https://your-app.onrender.com
  • Railway — connect GitHub repo, auto-deploy on every push, generous free tier
  • Google Cloud Run — deploy from GitHub or Docker, free when idle (scales to zero)

All three support environment variables in their dashboard — set VERIFY_TOKEN, WHATSAPP_TOKEN, and PHONE_NUMBER_ID there, not in your code.

2

Configure webhook in Meta

  1. Developer App → WhatsApp → Configuration → Webhook → Edit
  2. Set Callback URL to https://your-server.com/webhook
  3. Set Verify Token to the same string you put in your VERIFY_TOKEN env var
  4. Click Verify and Save — Meta sends a GET to your webhook immediately
  5. Your server should log "Webhook verified successfully" and Meta shows a green checkmark
  6. Under Webhook fields, make sure messages is toggled to Subscribed
3

Test it end-to-end

  1. From one of your 5 registered test recipient numbers, send a WhatsApp message to your bot's number
  2. Check your server logs — you should see the incoming payload
  3. The bot should reply: "You said: [your message]"

If this works, your entire integration is functioning correctly. You're ready for the final step.

Tip: deploy early, not last

Don't wait until everything is working locally before deploying. Push to Render, Railway, or Cloud Run early — even with a basic echo server — so you have a stable HTTPS URL to register with Meta from the start. All three platforms redeploy automatically on every GitHub push, so your changes go live in under a minute.
