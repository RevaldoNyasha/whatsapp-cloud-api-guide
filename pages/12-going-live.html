Going Live

Your app starts in Development mode, where only 5 registered test recipients can interact with it. Switching to Live mode opens it to any WhatsApp user in the world. Here's how to do it safely.

Development mode vs Live mode

Development ModeLive Mode
Who can message your botOnly your 5 registered test recipientsAny WhatsApp user
Messages from other numbersSilently ignored — no webhook eventForwarded to your webhook
Display name requiredNoNo — you can go live without it (cosmetic limitation only)
Business verification requiredNoStrongly recommended
App Review requiredNoMay be required for some permissions

Pre-flight checklist

Before switching to Live mode, go through this checklist. Click each item to mark it complete.

  • Webhook URL is configured and Meta shows a green checkmark
  • The messages webhook field is subscribed (not just the webhook itself)
  • PHONE_NUMBER_ID env variable points to your permanent number, not the test number
  • WHATSAPP_TOKEN is a System User Token (never expires) — not the 24-hour temporary token
  • Display name submitted (approval is not required to go Live — see Section 6)
  • Business portfolio verified in Meta Business Suite
  • WABA subscribed to your app (POST /{WABA_ID}/subscribed_apps returned { "success": true })
  • End-to-end test passed: sent a message from a test number, bot replied correctly
  • Server responds HTTP 200 within 5 seconds for all webhook payloads
  • .env file is in .gitignore and token is not exposed in source code
1

Switch the app mode

  1. Go to developers.facebook.com/apps and open your app
  2. At the top of any page in your app dashboard, find the App Mode toggle — it currently shows Development
  3. Click it to switch to Live
  4. A confirmation dialog appears — confirm

Your app is now live. Any WhatsApp user who messages your registered number will have their message forwarded to your webhook.

2

App Review (if prompted)

After switching to Live, Meta may prompt you to submit an App Review for the whatsapp_business_messaging permission.

App Review is straightforward for basic messaging

For a standard bot that sends and receives messages for your business users, App Review is simple. Describe what your app does plainly ("A chatbot that allows customers to contact our support team via WhatsApp"). Approval typically takes a few business days. You don't need a screencast for basic messaging permissions.

3

One thing to skip: Tech Provider / BSP

Somewhere in the Meta dashboard you may see a prompt to "Become a Tech Provider" or "Business Solution Provider (BSP)".

Skip the Tech Provider option

A BSP/Tech Provider is only for companies that build WhatsApp solutions on behalf of other businesses — agencies, resellers, platform builders. If you're building your own bot for your own business, you do not need this. Opting in adds significant compliance requirements and a lengthy onboarding process for no benefit in your case.

4

Verify end-to-end in Live mode

  1. Have a number that was not in your 5 test recipients send a message to your bot
  2. Check your server logs — you should see the incoming payload with the real number in the from field
  3. Your bot should reply correctly

If this works, you're done. Your WhatsApp integration is live and open to the public.

Top 5 things that break after going live

Common post-launch problems and fixes
#SymptomRoot causeFix
1 Bot receives messages but never replies Access token expired (OAuthException 190) Switch to System User Token (Section 7)
2 Messages from permanent number never arrive at webhook WABA not subscribed to the app Run POST /{WABA_ID}/subscribed_apps again
3 Bot replies but from the test number, not the real number Wrong PHONE_NUMBER_ID in env vars Update PHONE_NUMBER_ID to your permanent number's ID
4 No POST requests reach the server at all messages webhook field was unsubscribed App → WhatsApp → Configuration → re-subscribe messages field
5 Messages from real users silently ignored App mode reverted to Development Check the App Mode toggle at the top of the developer dashboard
