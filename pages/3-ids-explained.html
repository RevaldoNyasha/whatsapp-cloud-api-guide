Understanding IDs

Meta gives you at least two different IDs in the WhatsApp dashboard. They look similar, they're both long numbers, and they're used for completely different purposes. Mixing them up is one of the most common bugs.

The two IDs — what they are

Phone Number IDWABA ID
Full name Phone Number ID WhatsApp Business Account ID
What it identifies A specific phone number registered under your account The container account that holds one or more phone numbers
Used in API calls to send or receive messages Account-level operations: subscribing apps, creating templates, fetching business info
Example endpoint POST /{PHONE_NUMBER_ID}/messages POST /{WABA_ID}/subscribed_apps
In your .env file PHONE_NUMBER_ID=... (required) Used in Postman calls, not usually in app env vars

The hierarchy

Think of it as a tree. Your Meta App contains one or more WABAs, and each WABA contains one or more phone numbers. Each level has its own ID:

Your Meta Developer App │ └── WhatsApp Business Account (WABA) ID: 1684001316178308 ←── WABA ID │ ├── Phone Number: +1 555 140 3562 (Test, provided by Meta) │ ID: 987654321098765 ←── Phone Number ID │ └── Phone Number: +263 77 555 0000 (Your permanent number) ID: 1043096775563494 ←── Phone Number ID
Watch out — two numbers, two IDs

If you have both a test number and your own permanent number, each has its own Phone Number ID. Your PHONE_NUMBER_ID environment variable must point to the correct one — the permanent number you actually want to use in production, not the test number.

Messages sent using the wrong Phone Number ID will go out from the wrong number, which can be very confusing during testing.

Where to find each ID

1

Finding the Phone Number ID

  1. Go to developers.facebook.com/apps and open your app.
  2. In the left sidebar, click WhatsApp → API Setup.
  3. Under the "From" section, if you have multiple numbers use the dropdown to select the right one.
  4. The Phone Number ID is shown directly below the selected phone number.
2

Finding the WABA ID

There are two places to find it:

Option A — In the Developer App:

  1. Same page: WhatsApp → API Setup.
  2. The WhatsApp Business Account ID is shown at the top of the page, above the phone numbers section.

Option B — In Meta Business Suite:

  1. Go to business.facebook.com.
  2. Click the gear icon (Settings) → Business Settings.
  3. In the left menu, under "Accounts", click "WhatsApp Accounts".
  4. The number listed next to each WABA name is the WABA ID.

Real-world example: send a text message

This is the full API call to send a text message. Notice which ID goes where:

HTTP 
POST https://graph.facebook.com/v22.0/{PHONE_NUMBER_ID}/messages
Authorization: Bearer {ACCESS_TOKEN}
Content-Type: application/json

{
  "messaging_product": "whatsapp",
  "to": "263775550000",
  "type": "text",
  "text": { "body": "Hello from the API!" }
}

The {PHONE_NUMBER_ID} in the URL is the ID of your number (the sender). The to field is the recipient's phone number in international format without the + sign.

Save both IDs now

Copy both your WABA ID and your Phone Number ID into a secure notes file right now. You'll need the WABA ID for the subscription call in Section 5, and the Phone Number ID in your app's .env file.
